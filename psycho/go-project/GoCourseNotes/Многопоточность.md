при одном ядре одновременно ничего не работает, всё работает последовательно(одна переключение контекста и вторая прога)
поток (thread) - последовательность операций, выполнением которых на процессе управляет ОС
у каждого процесса есть как минимум один главный поток, при этом прога может запускать доп потоки и параллельно производить вычисления, все потоки в рамках одного процесса делят ресурсы между собой 
состояние гонки, когда логика выполнения проги зависит от порядка выполнения кода в разных потоках(один поток меняет данные, другой читает их )
асинхронность работает в рамках одного процесса, потока
