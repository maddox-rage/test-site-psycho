## ООП

1. Как в Go реализуется инкапсуляция? (Уровень пакетов, экспорт имён). инкапсуляция реализуется на уровне пакетов через экспорт имён: заглавная буква - экспорт, видны извне пакета; со строчной - только внутри. это скрывает реализацию, позволяя экспортировать только api, а не поля структур напрямую. методы на экспортированных типах контролируют доступ к приватным полям.

2. Чем композиция в Go отличается от классического наследования? композиция использует embedding структур. оно делегирует методы вложенному типу без иерархии, избегая хрупкой базы.

3. Что такое "Shadowing" полей при встраивании (embedding) структур? shadowing полей при встраивании происходит, когда внешняя структура имеет поле с тем же именем, что и вложенная: внешнее перекрывает внутреннее. это позволяет переопределять поле без конфликтов, сохраняя доступ к оригиналу через квалификацию.

4. Могут ли методы быть определены для не-структурных типов? Каковы ограничения? да, могут, но только в том же пакете, где тип определён или базовый пакет локальный. ограничения: нельзя добавлять метода к типам из других пакетов, чтобы избежать неявных зависимостей.

5. Как реализовать паттерн "Конструктор" в Go? Почему принято возвращать интерфейс или структуру? функция NewT(), возвращает указатель или интерфейс для инициализации; интерфейс для абстракции, структуры для простоты, указатель чтобы избежать копирование и разрешить мутацию

7. Как в Go реализовать полиморфизм без наследования? с помощью интерфейсов, любой тип реализующий методы интерфейса, используется полиморфно

8. Почему в Go нет перегрузки методов (method overloading)?чтобы избежать неоднозначности, сигнатуры определяются ресивером + именем, параметры не влияют,

9. Как работает struct{} (пустая структура) и почему она занимает 0 байт? потому что там нет полей, полезно для сигналов в мапу или сет без памяти; размер сумма полей плюс паддинг

10. Влияет ли порядок полей в структуре на её размер? влияет из-за выравнивания, компилятор добавляет байты для выравнивания, для оптимизации лучше сортировать по убыванию

11. Можно ли расширить функционал типа из чужого пакета, добавив ему метод? нет, если это не локальный базовый, решений: интерфейс или композиция с новым типом
